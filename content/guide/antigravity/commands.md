안티그래비티는 복잡한 CLI(명령줄 인터페이스) 명령어보다는 **자연어 지시**와 설정 파일을 통한 제어가 핵심입니다.

---

## A. 핵심 자연어 명령어 (채팅창 입력)

비개발자는 채팅창에 다음과 같은 형식으로 명령합니다.

### 1. 기능 구현

**명령어 예시:**
```
"이 함수만 고쳐줘"
"로그인 페이지 만들어줘"
"상품 목록을 카드 형태로 보여줘"
```

**팁:**
범위를 좁혀서 지시하는 것이 오류를 줄입니다.

❌ **나쁜 예시:**
```
"웹사이트 만들어줘"
→ 너무 광범위해서 원하는 결과가 안 나올 수 있음
```

✅ **좋은 예시:**
```
"쇼핑몰 메인 페이지를 만들어줘.
상단에 로고와 검색창, 중간에 추천 상품 3개, 하단에 푸터"
→ 구체적인 요구사항으로 정확한 결과 도출
```

---

### 2. 이미지 생성

내장된 **Nanobana 모델**이 이미지를 생성합니다.

**명령어 예시:**
```
"해저를 탐험하는 유튜브 썸네일 만들어줘"
"로고 만들어줘. 파란색 배경에 로켓 모양"
"404 에러 페이지용 일러스트 그려줘"
```

**생성된 이미지 위치:**
```
/project-folder/assets/images/
→ 자동으로 저장되며 코드에서 참조됨
```

**고급 옵션:**
```
"로고 만들어줘.
- 스타일: 미니멀리즘
- 색상: 파란색 계열
- 크기: 512x512px
- 배경: 투명"
```

---

### 3. 브라우저 제어

AI가 직접 웹 브라우저를 실행하여 **UI 조작을 수행**합니다.

**명령어 예시:**
```
"웹사이트 열어서 회원가입 테스트해줘"
"로그인 버튼 눌러보고 제대로 작동하는지 확인해줘"
"장바구니에 상품 3개 넣고 결제까지 진행해봐"
```

**동작 과정:**
```
1. AI가 Chrome 브라우저 실행
2. localhost:3000 (또는 지정된 URL) 접속
3. 버튼 클릭, 폼 입력 등 자동 수행
4. 스크린샷 캡처하여 결과 보고
5. 오류 발견 시 자동 수정
```

**활용 사례:**
- E2E 테스트 자동화
- 회원가입 플로우 검증
- 결제 프로세스 확인
- UI/UX 버그 탐지

---

### 4. 버전 관리 (Git)

깃(Git)을 몰라도 AI가 **버전 관리**를 대신 수행합니다.

**명령어 예시:**
```
"지금 상태 커밋(Commit) 해줘"
"이전 버전으로 되돌려줘"
"변경 이력 보여줘"
```

**실제 동작:**
```
사용자: "지금 상태 커밋해줘"

AI 수행:
  git add .
  git commit -m "로그인 페이지 UI 개선"

결과: "변경사항이 저장되었습니다 (커밋 ID: a3f2b1c)"
```

---

### 5. 에러 수정

터미널 로그를 분석하여 **자동으로 수정**합니다.

**명령어 예시:**
```
"방금 발생한 에러 로그 확인해서 고쳐줘"
"npm install 오류 해결해줘"
"TypeError가 계속 나는데 원인 찾아줘"
```

**동작 과정:**
```
1. 터미널 에러 로그 읽기
2. 에러 원인 분석
3. 관련 파일 수정
4. 테스트 실행으로 검증
5. 수정 완료 보고
```

## B. 명령어 패턴 가이드

### 명확한 지시어 사용

| 목적 | 명령어 패턴 | 예시 |
|------|-----------|------|
| **생성** | "~를 만들어줘" | "로그인 폼 만들어줘" |
| **수정** | "~를 바꿔줘/고쳐줘" | "버튼 색깔을 파란색으로 바꿔줘" |
| **삭제** | "~를 지워줘" | "이 기능 지워줘" |
| **확인** | "~를 테스트해줘" | "회원가입이 제대로 되는지 테스트해줘" |
| **설명** | "~를 설명해줘" | "이 코드가 뭐 하는 건지 설명해줘" |

---

### 범위 지정하기

**파일 단위:**
```
"login.js 파일에서 에러 처리 추가해줘"
```

**기능 단위:**
```
"장바구니 기능만 수정해줘"
```

**라인 단위:**
```
"27번째 줄의 함수 고쳐줘"
```

---

### 컨텍스트 제공하기

**배경 정보 포함:**
```
❌ "버튼 추가해줘"

✅ "로그인 페이지 하단에 '비밀번호 찾기' 버튼 추가해줘.
    파란색 배경, 흰색 글씨로"
```

**예시 제공:**
```
"네이버 쇼핑처럼 상품 카드를 만들어줘.
이미지 위에 제목, 가격, 별점이 보이게"
```

## C. 고급 명령어

### 멀티 태스크

여러 작업을 한 번에 지시:
```
"다음 작업을 순서대로 해줘:
1. 로그인 페이지 만들기
2. 회원가입 폼 추가
3. 비밀번호 찾기 기능 구현
4. 모든 페이지 연결하기"
```

---

### 조건부 명령

특정 조건에 따라 다르게 동작:
```
"로그인 성공하면 대시보드로 이동,
실패하면 에러 메시지 표시해줘"
```

---

### 스타일 지정

디자인 요구사항 포함:
```
"버튼을 만들어줘.
- 크기: 200px x 50px
- 색상: 그라데이션 (주황 → 빨강)
- 모서리: 둥글게 (10px)
- 호버 시: 약간 커지게"
```

---

## D. 명령어 실행 흐름

```
1. 채팅창에 명령어 입력
   ↓
2. AI가 명령어 분석
   ↓
3. Planning Mode인 경우: 계획 수립 → 승인 요청
   Fast Mode인 경우: 즉시 실행
   ↓
4. 코드 생성/수정
   ↓
5. 브라우저 자동 테스트 (필요 시)
   ↓
6. 결과 보고 및 확인
   ↓
7. 오류 발견 시: 자동 수정 후 재테스트
```

## F. 명령어 팁

<Callout type="tip" title="💡 효율적인 명령어 작성법">
1. **구체적으로**: "버튼" → "파란색 로그인 버튼"
2. **단계별로**: 복잡한 작업은 여러 단계로 나누기
3. **예시 포함**: 원하는 결과물 예시 제공
4. **검증 요청**: "~하고 테스트도 해줘" 추가
5. **피드백 제공**: 결과가 마음에 안 들면 "좀 더 밝게", "조금 작게" 등 세부 조정
</Callout>

<Callout type="warning" title="⚠️ 주의사항">
- 너무 긴 명령어는 여러 개로 나누기
- 모호한 표현 피하기 ("예쁘게", "멋지게" 등)
- 중요한 파일 수정 전에는 백업 요청하기
</Callout>
