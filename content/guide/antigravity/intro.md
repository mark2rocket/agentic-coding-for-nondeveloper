## 핵심 정의

구글 안티그래비티는 사용자의 자연어 명령을 코드로 변환하고, 실행, 디버깅, 배포까지 스스로 수행하는 **'에이전틱 코드 에디터'**입니다.

기존의 코딩 보조 도구(Copilot 등)가 코드 추천에 그쳤다면, 안티그래비티는 프로젝트 전체의 구조를 설계하고 브라우저를 직접 제어하여 테스트까지 수행합니다.

<Callout type="info" title="📅 출시 정보">
2025년 11월 구글이 제미나이(Gemini)와 함께 공식 출시한 에이전틱(Agentic) 코딩 통합 개발 환경(IDE)입니다.
</Callout>

---

## 누구를 위한 도구인가?

안티그래비티는 **비개발자도 자연어 명령을 통해 앱과 웹 서비스를 구축**할 수 있도록 설계되었습니다.

특히 다음과 같은 기능에 특화되어 있습니다:
- 브라우저 제어
- 멀티 에이전트 조율
- 이미지 자동 생성
- 자동 테스트 및 디버깅

---

## 주요 특징

### 1. Gemini 3.0 Pro 탑재

구글의 최신 모델인 **Gemini 3.0 Pro**(High/Low) 및 **Gemini 3.0 Deep**(추론용)을 기반으로 작동하며, 긴 맥락(Context)을 이해합니다.

**장점:**
- 복잡한 프로젝트 요구사항도 한 번에 파악
- 이전 대화 내용을 기억하며 일관된 작업 수행
- 여러 파일 간 관계를 이해하고 수정

---

### 2. 브라우저 에이전트

**AI가 웹 브라우저를 직접 실행**하여 결과물을 확인하고, 오류를 스스로 찾아 수정합니다.

**예시:**
```
"웹사이트 열어서 버튼 눌러봐"
→ AI가 실제로 브라우저를 실행하고 버튼 클릭
→ 작동 여부 확인 후 문제 발견 시 자동 수정
```

**활용 사례:**
- 회원가입 플로우 자동 테스트
- 결제 프로세스 검증
- UI 버그 자동 탐지

---

### 3. 멀티 모달 통합 (Nanobana)

이미지 생성 모델인 **'Gemini Nanobana'**가 내장되어 있어, 코딩 중 필요한 이미지를 즉시 생성할 수 있습니다.

**예시:**
```
"로고 만들어줘"
→ AI가 프로젝트 컨셉에 맞는 로고 이미지 생성
→ 자동으로 프로젝트 assets 폴더에 저장
```

**활용 사례:**
- 앱 아이콘 생성
- 유튜브 썸네일 제작
- UI 목업 이미지 생성

---

### 4. 에이전트 매니저

**여러 개의 프로젝트(워크스페이스)를 동시에 띄워두고** 각기 다른 작업을 수행하는 에이전트들을 관리할 수 있습니다.

**예시:**
```
워크스페이스 1: "쇼핑몰 프론트엔드 작업"
워크스페이스 2: "관리자 대시보드 작업"
워크스페이스 3: "API 서버 작업"

→ 3개 에이전트가 동시에 작업 수행
→ 서로 다른 기술 스택과 컨텍스트 유지
```

## 기존 도구와의 차이점

| 구분 | GitHub Copilot | Cursor | 안티그래비티 |
|------|---------------|--------|------------|
| **역할** | 코드 자동완성 | AI 페어 프로그래밍 | 자율 개발 에이전트 |
| **작업 범위** | 함수/라인 단위 | 파일/기능 단위 | 프로젝트 전체 |
| **브라우저 제어** | ❌ | ❌ | ✅ |
| **이미지 생성** | ❌ | ❌ | ✅ (Nanobana) |
| **자동 테스트** | ❌ | ❌ | ✅ |
| **멀티 에이전트** | ❌ | ❌ | ✅ |

---

## 안티그래비티가 적합한 경우

<Callout type="success" title="✅ 이런 분들에게 추천">
- 코딩을 전혀 모르지만 아이디어를 구현하고 싶은 사람
- 반복적인 테스트 작업을 자동화하고 싶은 개발자
- 여러 프로젝트를 동시에 관리해야 하는 팀
- 브라우저 자동화가 필요한 QA 작업
- 이미지 리소스가 많이 필요한 프로젝트
</Callout>

<Callout type="warning" title="⚠️ 주의사항">
안티그래비티는 강력한 도구이지만, **터미널 명령 자동 실행 설정**을 잘못하면 파일이 삭제될 위험이 있습니다. 반드시 설정을 확인하고 사용해야 합니다 (자세한 내용은 "3-6. 보안 및 주의사항" 참고).
</Callout>
