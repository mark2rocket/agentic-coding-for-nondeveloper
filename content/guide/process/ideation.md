---
title: "1. 기획 및 요구사항 정의 (Ideation)"
order: 1
---

**설명**: 코드를 직접 짜는 것이 아니라, AI에게 '무엇을 만들고 싶은지' 명확한 의도(Vibe)를 전달하는 가장 중요한 첫 단계입니다. 개발 지식보다는 서비스의 논리적인 흐름과 사용자의 경험을 설명하는 것이 핵심입니다.

**예시**:

```
"사용자가 '기분'을 입력하면 그에 맞는 음악을 추천해주는 웹사이트를 만들어줘.
디자인은 다크 모드에 네온 컬러를 써서 힙한 분위기를 내주고,
추천 결과는 유튜브 링크로 바로 연결되게 해줘."
```

**부연 설명**: 비개발자가 코딩에 실패하는 가장 큰 이유는 기술이 부족해서가 아니라, AI에게 무엇을 시킬지 모호하게 말하기 때문입니다. 단순히 "음악 추천 사이트 만들어줘"라고 하면 AI는 가장 일반적이고 재미없는 결과를 내놓습니다. 어떤 기능이 필요한지(로그인, 결제, 게시판 등), 어떤 디자인 스타일을 원하는지, 주 사용자는 누구인지 구체적으로 묘사해야 합니다. 마치 건축가에게 설계도를 말로 설명해주듯 구체적일수록 수정 횟수가 획기적으로 줄어듭니다.

**한줄 요약**: AI에게 '어떻게'가 아니라 '무엇'을 만들지, 최대한 구체적인 상황극을 하듯 설명하세요.

---

## 🎯 클로드코드로 5분 실습하기

<Callout type="success" title="✅ 실습 목표">
좋은 기획 vs 나쁜 기획의 차이를 직접 체험하고, AI가 어떻게 다르게 반응하는지 확인합니다.
</Callout>

### Step 1: 나쁜 기획 예시 (모호한 요청)

터미널에서 클로드코드를 실행하고 다음과 같이 요청해보세요:

```bash
claude code
```

채팅창에 입력:
```
할 일 관리 앱 만들어줘
```

**결과**: AI가 질문을 많이 하거나, 매우 기본적인 기능만 만듭니다.

---

### Step 2: 좋은 기획 예시 (구체적 요청)

이번엔 같은 앱을 구체적으로 요청해보세요:

```
할 일 관리 웹앱을 만들어줘. 요구사항은 다음과 같아:

기능:
- 할 일 입력창 (텍스트 입력)
- 추가 버튼 클릭 시 리스트에 추가
- 각 할 일 옆에 체크박스 (완료 표시)
- 완료된 항목은 회색 줄 그어지기
- 삭제 버튼으로 항목 제거
- localStorage에 저장해서 새로고침해도 유지

디자인:
- 배경: 연한 파스텔 톤
- 중앙 정렬된 카드 형태
- 입력창과 버튼은 파란색 계열
- 모바일에서도 잘 보이게 반응형

기술 스택:
- 순수 HTML, CSS, JavaScript (프레임워크 없이)
- 파일 구조: index.html, style.css, script.js
```

**결과**: AI가 질문 없이 바로 정확한 결과물을 만듭니다.

---

### Step 3: 두 결과 비교하기

**나쁜 기획의 문제점:**
- AI가 추가 질문을 많이 함
- 기본적인 기능만 구현됨
- 디자인이 투박함
- 데이터 저장 기능 누락

**좋은 기획의 장점:**
- 한 번에 원하는 결과 획득
- 모든 기능이 포함됨
- 깔끔한 디자인
- 데이터 지속성 확보

---

### 💡 실습에서 배운 점

<Callout type="tip" title="기획 체크리스트">
AI에게 요청할 때 다음 4가지를 반드시 포함하세요:

1. **기능 목록**: 사용자가 뭘 할 수 있는가?
2. **디자인 방향**: 어떤 느낌? 색상? 레이아웃?
3. **기술 요구사항**: 어떤 도구로 만들까? (모르면 생략 가능)
4. **특수 조건**: 모바일 대응? 오프라인 동작? 빠른 속도?
</Callout>

**소요 시간**: 5분

## 🚀 GPTs/Gemini로 PRD 자동 생성하기

<Callout type="success" title="💎 PRD 젬 활용">
추상적인 아이디어를 AI 코딩 에이전트가 바로 구현할 수 있는 완벽한 PRD(제품 요구사항 정의서)로 변환해주는 전문 프롬프트입니다.
</Callout>

### 사용 방법

1. **ChatGPT Plus** 또는 **Google Gemini**에 접속
2. 아래 프롬프트를 **전체 복사**하여 새 대화 시작
3. 당신의 아이디어를 입력하면 3가지 옵션 제안
4. 옵션 선택 후 완성된 PRD.md 파일 생성

---

### PRD 젬 프롬프트 (전체 복사)

```text
당신은 20년 차 시니어 프로덕트 매니저(Senior Product Manager)이자, '바이브코딩(AI를 활용한 자연어 기반 코딩)' 프로젝트의 설계 전문가인 'PRD 젬'입니다. 사용자의 추상적인 아이디어를 AI 코딩 에이전트(Cursor, Replit 등)가 즉시 구현할 수 있는 수준의 완벽한 PRD(제품 요구사항 정의서)로 구체화하는 것이 당신의 사명입니다.

[핵심 원칙]
* Structured Thinking: 모든 답변은 구조화되어야 하며, 논리적 비약이 없어야 합니다.
* Tech-Driven: 개발 구현 가능성을 최우선으로 고려합니다.
* User-Centric: 사용자의 행동 흐름(Flow)을 중심으로 기능을 정의합니다.

[진행 프로세스 및 단계별 지침]

1단계: 기능 제안 및 유저 플로우 설계 (Options & Flows)
사용자의 아이디어를 분석하여 다음 3가지 전략적 옵션을 제안하십시오. 각 옵션은 '사용자 경험의 깊이'에 따라 구분되어야 합니다.
* 옵션 A (MVP 집중형): 핵심 문제 해결을 위한 최소 기능. 군더더기 없는 직관적인 플로우. 빠른 배포 목표.
* 옵션 B (시장 경쟁력 강화형): 리텐션(Retention)과 인게이지먼트(Engagement)를 고려한 기능 및 UX 추가.
* 옵션 C (혁신/기술 주도형): AI 자동화, 개인화 알고리즘 등 최신 기술 트렌드를 접목한 고도화된 플로우.

각 옵션별 필수 출력 항목:
1. 컨셉 요약: 한 줄 정의.
2. 예상 구현 난이도: (상/중/하) 및 구체적인 기술적 이유.
3. 핵심 유저 플로우 (Core User Flow): 구체적인 행동 단계를 '[단계] -> [단계]' 형식으로 작성.
4. 주요 장점: 비즈니스 및 사용자 관점의 이점 기술.
5. 추천 대상: 해당 옵션이 적합한 구체적 상황 제시.
* 답변 마지막에는 사용자가 옵션을 선택하거나 수정 요청을 하도록 유도하십시오.

2단계: PRD.md 파일 작성 (PRD Generation)
사용자가 옵션을 결정하면, 개발 에이전트가 즉시 이해할 수 있는 'PRD.md' 형식의 마크다운 문서를 작성하십시오.
* 필수 포함 항목: 프로젝트명 및 버전, 문제 정의 및 목표, 타겟 유저 및 페르소나, User Stories (Acceptance Criteria 포함), 상세 유저 플로우 (화면 단위), Functional Requirements (데이터 구조, API, 로직, 예외 처리), UI/UX Guidelines, Tech Stack Recommendation (Next.js, Supabase, Tailwind CSS 등 최적화된 스택).

3단계: 기능 작동 시뮬레이션 (Simulation & Walkthrough)
PRD 작성이 완료되면, 현재 시제의 구어체로 사용자가 결과물을 시연하듯 논리적 인과 관계를 설명하십시오.

[제약 사항 및 어조]
* 20년 차 PM의 전문적이고 날카로우며 신뢰감 있는 톤을 유지하십시오.
* 모든 PRD는 복사 가능한 '마크다운 코드 블록'으로 제공하십시오.
* '적절히', '잘' 같은 모호한 표현을 금지하고 데이터 필드명, 상태 값, 구체적 수치를 명시하십시오.
```

---

### 활용 예시

**입력:**
```
음악 추천 웹사이트를 만들고 싶어요. 사용자의 기분을 입력하면 그에 맞는 음악을 추천해주는 서비스입니다.
```

**출력:**
- 3가지 옵션 제안 (MVP / 시장 경쟁력 / 혁신형)
- 선택한 옵션의 완벽한 PRD.md 파일
- 구현 가능한 상세 유저 플로우
- 기술 스택 추천

---

### PRD를 클로드코드에 적용하기

1. PRD.md 파일을 프로젝트 루트에 저장
2. 클로드코드 실행: `claude code`
3. PRD 파일 읽기: "PRD.md 파일을 읽고 이대로 구현해줘"
4. AI가 PRD 기반으로 완벽한 코드 생성

<Callout type="tip" title="💡 PRD 작성의 장점">
- ✅ 개발 시작 전 명확한 요구사항 정리
- ✅ AI가 질문 없이 바로 구현 가능
- ✅ 팀원과 공유 가능한 문서
- ✅ 수정 및 확장 용이
</Callout>
