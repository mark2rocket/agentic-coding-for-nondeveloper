## 기획이란? -- "AI에게 주는 설계도"

<Callout type="success" title="건축가 비유로 이해하기">

집을 지을 때를 생각해보세요.

**❌ 나쁜 의뢰:**
"집 지어주세요."

**✅ 좋은 의뢰:**
"침실 3개, 거실 1개, 주방은 오픈형으로, 마당은 남향으로, 북유럽 스타일로 지어주세요."

AI 코딩도 똑같습니다!
- 구체적일수록 = 수정 횟수가 줄어듭니다
- 막연할수록 = 계속 "이게 아닌데..." 반복

</Callout>

**이 단계의 핵심:** 코드를 직접 짜는 것이 아니라, AI에게 **'무엇을 만들고 싶은지'** 명확하게 전달하는 것입니다.

---

## 왜 기획이 중요한가?

<LevelBox>

**비개발자가 AI 코딩에 실패하는 가장 큰 이유:**

기술이 부족해서가 **아니라**, AI에게 무엇을 시킬지 **모호하게** 말하기 때문입니다.

**예시:**

❌ **모호한 요청:** "음악 추천 사이트 만들어줘"
- AI는 가장 일반적이고 재미없는 결과를 내놓습니다
- 질문을 계속 하거나, 엉뚱한 기능을 만듭니다

✅ **구체적 요청:**
"사용자가 '기분'을 입력하면 그에 맞는 음악을 추천해주는 웹사이트를 만들어줘.
디자인은 어두운 배경에 네온 불빛 같은 밝은 색상으로 힙한 분위기를 내고,
추천 결과는 유튜브 링크로 바로 재생할 수 있게 해줘."

- AI가 한 번에 정확히 이해하고 만듭니다

</LevelBox>

**핵심:** 마치 건축가에게 설계도를 말로 설명해주듯 **구체적일수록** 수정 횟수가 획기적으로 줄어듭니다.

## 🎯 클로드코드로 7분 실습하기

<Callout type="success" title="실습 목표">
좋은 기획 vs 나쁜 기획의 차이를 직접 체험하고, AI가 어떻게 다르게 반응하는지 확인합니다.
</Callout>

### Step 1: 나쁜 기획 예시 (모호한 요청)

터미널에서 클로드코드를 실행하고 다음과 같이 요청해보세요:

```bash
claude code
```

채팅창에 입력:

```text
할 일 관리 앱 만들어줘
```

**결과**: AI가 질문을 많이 하거나, 매우 기본적인 기능만 만듭니다.

---

### Step 2: 좋은 기획 예시 (구체적 요청)

이번엔 같은 앱을 구체적으로 요청해보세요:

<Callout type="info" title="용어 설명">

아래 예시에 나오는 용어들을 미리 알아두세요:

- **브라우저 저장소**: 브라우저를 닫았다 열어도 데이터가 그대로 있게 하는 기능
- **반응형**: 컴퓨터, 휴대폰, 태블릿 어디서든 화면 크기에 맞게 잘 보이는 것
- **가장 기본적인 웹 기술**: 복잡한 도구 없이 HTML, CSS, JavaScript만 사용

</Callout>

```text
할 일 관리 웹앱을 만들어줘. 요구사항은 다음과 같아:

기능:
- 할 일을 입력할 수 있는 텍스트 상자
- "추가" 버튼을 누르면 리스트에 추가됨
- 각 할 일 옆에 완료 표시할 수 있는 체크박스
- 완료된 항목은 회색에 줄 그어지기
- "삭제" 버튼으로 항목 제거
- 브라우저 저장소에 저장해서 새로고침해도 데이터 유지

디자인:
- 배경: 연한 파스텔 톤 (부드러운 색상)
- 중앙 정렬된 카드 모양
- 입력창과 버튼은 파란색 계열
- 휴대폰, 태블릿, 컴퓨터 어디서든 잘 보이게

기술 방식:
- 가장 기본적인 웹 기술로 만들기 (복잡한 도구 사용 안 함)
- 파일 3개로 구성: index.html, style.css, script.js
```

**결과**: AI가 질문 없이 바로 정확한 결과물을 만듭니다.

### Step 2.5: 결과물 확인하기

<LevelBox>

**1. 생성된 파일 찾기**

터미널에서 다음 명령어 입력:
```bash
ls
```

화면에 다음 파일들이 보이면 성공:
- `index.html`
- `style.css`
- `script.js`

**2. 브라우저에서 열기**

**macOS 사용자:**
```bash
open index.html
```

**Windows 사용자:**
```bash
start index.html
```

**3. 작동 확인 체크리스트**

브라우저가 열리면 다음을 테스트해보세요:

- [ ] 텍스트 입력하고 "추가" 버튼 클릭 → 리스트에 추가되나요?
- [ ] 체크박스 클릭 → 완료 표시되고 줄 그어지나요?
- [ ] "삭제" 버튼 클릭 → 항목이 사라지나요?
- [ ] 할 일을 여러 개 추가하고 새로고침(F5) → 데이터가 유지되나요?
- [ ] 휴대폰 크기로 브라우저 창을 줄여보기 → 잘 보이나요?

**모두 체크되면 성공!** ✅

</LevelBox>

<Callout type="tip" title="스크린샷 찍어두기">

**추천:** 결과물의 스크린샷을 찍어두세요!

- macOS: `Cmd + Shift + 4`
- Windows: `Win + Shift + S`

나중에 포트폴리오나 SNS에 공유할 때 유용합니다.

</Callout>

### Step 3: 두 결과 비교하기

**나쁜 기획의 문제점:**
- AI가 추가 질문을 많이 함
- 기본적인 기능만 구현됨
- 디자인이 투박함
- 데이터 저장 기능 누락

**좋은 기획의 장점:**
- 한 번에 원하는 결과 획득
- 모든 기능이 포함됨
- 깔끔한 디자인
- 데이터 지속성 확보

### 💡 실습에서 배운 점

<Callout type="tip" title="기획 체크리스트">
AI에게 요청할 때 다음 4가지를 반드시 포함하세요:

1. **기능 목록**: 사용자가 뭘 할 수 있는가?
2. **디자인 방향**: 어떤 느낌? 색상? 레이아웃?
3. **기술 요구사항**: 어떤 도구로 만들까? (모르면 생략 가능)
4. **특수 조건**: 모바일 대응? 오프라인 동작? 빠른 속도?
</Callout>

**소요 시간**: 5분

## 왜 PRD를 작성해야 하나요?

<Callout type="warning" title="PRD 없이 시작하면? (실제 사례)">

**시나리오: "할 일 관리 앱 만들어줘"**

```
09:00 - AI가 만듦 → "아 이건 아닌데... 디자인이 별로야"
09:30 - "디자인 바꿔줘" → AI가 처음부터 다시 만듦
10:00 - "기능 추가해줘" → 이전 코드랑 충돌, 에러 발생
10:30 - "아... 그냥 처음부터 다시..." → 좌절
11:00 - 5번째 반복 중... "AI 코딩은 안 돼" 포기
```

**결과:**
- ⏰ 3시간 소모
- 😫 좌절감 증가
- 💔 "AI 코딩은 나한테 안 맞아" 포기

</Callout>

<Callout type="success" title="PRD 작성 후 시작하면? (현실)">

**시나리오: PRD.md 먼저 작성**

```
09:00 - PRD 작성 시작 (ChatGPT 도움 받음)
09:20 - PRD 완성 및 검토
09:25 - 클로드 코드에 PRD 전달
09:35 - AI가 한 번에 정확히 구현
09:45 - 사소한 색상 수정 1회
09:50 - 완성!
```

**결과:**
- ⏰ **50분** 만에 완성
- 😊 만족도 높음
- 🎉 "AI 코딩 신세계!" 감탄

</Callout>

### PRD란? (Product Requirements Document)

<LevelBox>

**일상 용어로 설명:**

PRD는 "**제품 설계도**"입니다.

- 집을 짓기 전 건축가가 그리는 **청사진**
- 요리하기 전 보는 **레시피**
- 여행 가기 전 짜는 **여행 계획표**

**PRD에 들어가는 내용:**

1. **무엇을 만들까?** (목표)
2. **누가 쓸까?** (사용자)
3. **어떤 기능이 있나?** (기능 목록)
4. **어떻게 생겼나?** (디자인)
5. **어떻게 작동하나?** (사용자 경험)

</LevelBox>

<Callout type="info" title="PRD 작성의 3가지 장점">

1. **시간 절약**: 처음부터 명확하면 수정 횟수가 1/10로 줄어듭니다
2. **품질 향상**: AI가 전체 그림을 보고 일관되게 만듭니다
3. **협업 용이**: 팀원이나 친구와 공유하기 쉽습니다

</Callout>

## 🚀 GPTs/Gemini로 PRD 자동 생성하기

<Callout type="success" title="PRD 젬이란?">

추상적인 아이디어를 AI 코딩 에이전트가 바로 구현할 수 있는 완벽한 **PRD(제품 설계도)**로 변환해주는 마법 같은 프롬프트입니다.

**쉽게 말하면:**
- 당신: "음악 추천 사이트 만들고 싶어"
- PRD 젬: "알겠어요! 3가지 버전으로 상세 설계도 만들어드릴게요"

</Callout>

### 사용 방법

1. **ChatGPT Plus** 또는 **Google Gemini**에 접속
2. 아래 프롬프트를 **전체 복사**하여 새 대화 시작
3. 당신의 아이디어를 입력하면 3가지 옵션 제안
4. 옵션 선택 후 완성된 PRD.md 파일 생성

### 🎯 초보자용 PRD 젬 (추천)

처음 사용하시는 분은 이 간단한 버전을 사용하세요!

<LevelBox>

**전체 복사해서 ChatGPT/Gemini에 붙여넣기:**

```text
당신은 비개발자의 아이디어를 AI가 바로 만들 수 있는 상세한 설계도(PRD)로 바꿔주는 전문가입니다.

[작동 방식]
1. 사용자의 아이디어를 듣고 3가지 버전을 제안합니다:
   - 버전 A: 가장 간단한 버전 (핵심 기능만)
   - 버전 B: 표준 버전 (유용한 기능 추가)
   - 버전 C: 고급 버전 (최신 기술 활용)

2. 사용자가 버전을 선택하면 다음 내용이 담긴 PRD.md 파일을 만듭니다:
   - 프로젝트 이름과 목표
   - 누가 사용할까? (사용자)
   - 어떤 기능이 있나? (기능 목록)
   - 어떻게 생겼나? (디자인 방향)
   - 어떻게 작동하나? (사용 흐름)
   - 어떤 기술로 만들까? (추천 기술)

[중요한 규칙]
- 전문 용어를 쓰지 말고 일상 언어로 설명하기
- 모든 기능을 구체적으로 설명하기
- "적절히", "잘" 같은 모호한 표현 금지
- PRD는 복사 가능한 마크다운 코드 블록으로 제공

준비됐습니다! 어떤 걸 만들고 싶으신가요?
```

</LevelBox>

### 🔥 전문가용 PRD 젬 (상세 버전)

더 상세한 PRD가 필요하다면 이 버전을 사용하세요.

<LevelBox>

**전체 복사해서 ChatGPT/Gemini에 붙여넣기:**

```text
당신은 20년 차 시니어 프로덕트 매니저(Senior Product Manager)이자, '바이브코딩(AI를 활용한 자연어 기반 코딩)' 프로젝트의 설계 전문가인 'PRD 젬'입니다. 사용자의 추상적인 아이디어를 AI 코딩 에이전트(Cursor, Replit 등)가 즉시 구현할 수 있는 수준의 완벽한 PRD(제품 요구사항 정의서)로 구체화하는 것이 당신의 사명입니다.

[핵심 원칙]
* Structured Thinking: 모든 답변은 구조화되어야 하며, 논리적 비약이 없어야 합니다.
* Tech-Driven: 개발 구현 가능성을 최우선으로 고려합니다.
* User-Centric: 사용자의 행동 흐름(Flow)을 중심으로 기능을 정의합니다.

[진행 프로세스 및 단계별 지침]

1단계: 기능 제안 및 유저 플로우 설계 (Options & Flows)
사용자의 아이디어를 분석하여 다음 3가지 전략적 옵션을 제안하십시오. 각 옵션은 '사용자 경험의 깊이'에 따라 구분되어야 합니다.
* 옵션 A (MVP 집중형): 핵심 문제 해결을 위한 최소 기능. 군더더기 없는 직관적인 플로우. 빠른 배포 목표.
* 옵션 B (시장 경쟁력 강화형): 리텐션(Retention)과 인게이지먼트(Engagement)를 고려한 기능 및 UX 추가.
* 옵션 C (혁신/기술 주도형): AI 자동화, 개인화 알고리즘 등 최신 기술 트렌드를 접목한 고도화된 플로우.

각 옵션별 필수 출력 항목:
1. 컨셉 요약: 한 줄 정의.
2. 예상 구현 난이도: (상/중/하) 및 구체적인 기술적 이유.
3. 핵심 유저 플로우 (Core User Flow): 구체적인 행동 단계를 '[단계] -> [단계]' 형식으로 작성.
4. 주요 장점: 비즈니스 및 사용자 관점의 이점 기술.
5. 추천 대상: 해당 옵션이 적합한 구체적 상황 제시.
* 답변 마지막에는 사용자가 옵션을 선택하거나 수정 요청을 하도록 유도하십시오.

2단계: PRD.md 파일 작성 (PRD Generation)
사용자가 옵션을 결정하면, 개발 에이전트가 즉시 이해할 수 있는 'PRD.md' 형식의 마크다운 문서를 작성하십시오.
* 필수 포함 항목: 프로젝트명 및 버전, 문제 정의 및 목표, 타겟 유저 및 페르소나, User Stories (Acceptance Criteria 포함), 상세 유저 플로우 (화면 단위), Functional Requirements (데이터 구조, API, 로직, 예외 처리), UI/UX Guidelines, Tech Stack Recommendation (Next.js, Supabase, Tailwind CSS 등 최적화된 스택).

3단계: 기능 작동 시뮬레이션 (Simulation & Walkthrough)
PRD 작성이 완료되면, 현재 시제의 구어체로 사용자가 결과물을 시연하듯 논리적 인과 관계를 설명하십시오.
```

### PRD를 클로드코드에 적용하기

1. PRD.md 파일을 프로젝트 루트에 저장
2. 클로드코드 실행: `claude code`
3. PRD 파일 읽기: "PRD.md 파일을 읽고 이대로 구현해줘"
4. AI가 PRD 기반으로 완벽한 코드 생성

<Callout type="tip" title="PRD 작성의 장점">
- ✅ 개발 시작 전 명확한 요구사항 정리
- ✅ AI가 질문 없이 바로 구현 가능
- ✅ 팀원과 공유 가능한 문서
- ✅ 수정 및 확장 용이
</Callout>
