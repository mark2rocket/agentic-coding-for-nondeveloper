### Day 3-4: 도구 설치 & 기본 설정

**완전 초보 (안티그래비티):**
```
1. 공식 사이트에서 설치 파일 다운로드
2. 더블 클릭으로 설치
3. GUI 실행
4. 첫 프로젝트 생성
5. "Hello World" 만들어보기
```

**중급 (클로드코드):**
```
1. 터미널 기초 학습 (cd, ls, mkdir)
2. 클로드코드 설치
3. API 키 설정
4. 첫 명령어 실행
5. CLAUDE.md 작성 시작
```

**시간:** 2-3시간

**체크리스트:**
- [ ] 도구 설치 완료
- [ ] 기본 설정 완료
- [ ] 첫 명령어 실행 성공
- [ ] 에러 발생 시 해결 방법 익힘

### Day 5-7: 기본 명령어 익히기

**실습 과제:**
1. 간단한 HTML 페이지 생성
2. 버튼 추가하기
3. 버튼 클릭 시 메시지 표시
4. 스타일 변경 (색상, 크기)

**예시 프롬프트:**
```
"HTML 페이지 만들어줘. 제목은 '내 첫 페이지',
파란색 버튼 하나 추가해줘.
버튼 누르면 '안녕하세요!' 알림 뜨게 해줘."
```

**시간:** 하루 1시간씩

**체크리스트:**
- [ ] 파일 생성/수정/삭제
- [ ] 기본 HTML/CSS 이해
- [ ] AI와 대화하는 방법 익힘
- [ ] 에러 발생 시 AI에게 수정 요청하기

**1주차 완료 기준:**
✅ 도구 설치 및 기본 설정 완료
✅ 간단한 웹 페이지 만들 수 있음
✅ AI에게 요청하는 방법 익힘

## 2주차: 실전 프로젝트 만들기 🛠️

### 목표
- 실용적인 앱 3개 제작
- 문제 해결 능력 향상
- 코드 수정 및 커스터마이징

### Day 8-9: 프로젝트 1 - To-Do 앱

**목표:** 할 일 관리 앱 만들기

**기능:**
- [ ] 할 일 추가
- [ ] 할 일 완료 체크
- [ ] 할 일 삭제
- [ ] 로컬 저장 (새로고침해도 유지)

**프롬프트 예시:**
```
"To-Do 리스트 웹앱 만들어줘.
- 입력창에 할 일 입력하고 추가 버튼 누르면 리스트에 추가
- 체크박스 클릭하면 줄 그어지게
- 삭제 버튼으로 삭제
- localStorage 사용해서 새로고침해도 유지"
```

**학습 포인트:**
- 사용자 입력 처리
- 데이터 저장/불러오기
- 동적 UI 업데이트

**소요 시간:** 2-3시간

---

### Day 10-11: 프로젝트 2 - 계산기 앱

**목표:** 기본 계산기 만들기

**기능:**
- [ ] 사칙연산 (+, -, ×, ÷)
- [ ] 숫자 버튼 (0-9)
- [ ] 초기화 버튼
- [ ] 계산 결과 표시

**프롬프트 예시:**
```
"계산기 웹앱 만들어줘.
- 숫자 버튼 0-9
- 연산자 버튼 +, -, ×, ÷
- = 버튼 누르면 계산 결과
- C 버튼으로 초기화
- 디자인은 아이폰 계산기처럼"
```

**학습 포인트:**
- 이벤트 핸들링
- 수학 연산
- UI 디자인 개선

**소요 시간:** 2-3시간

---

### Day 12-14: 프로젝트 3 - 타이머 앱

**목표:** 뽀모도로 타이머 만들기

**기능:**
- [ ] 25분 작업 타이머
- [ ] 5분 휴식 타이머
- [ ] 시작/정지/리셋 버튼
- [ ] 알림 소리

**프롬프트 예시:**
```
"뽀모도로 타이머 만들어줘.
- 25분 카운트다운
- 시작, 일시정지, 리셋 버튼
- 0초 되면 알림 소리 재생
- 진행률 원형 그래프로 표시"
```

**학습 포인트:**
- 타이머 구현
- 오디오 재생
- 시각화 (프로그레스 바)

**소요 시간:** 3-4시간

**2주차 완료 기준:**
✅ 3개 프로젝트 완성
✅ 문제 발생 시 스스로 AI에게 요청해서 해결
✅ 코드 수정 및 커스터마이징 가능

## 3주차: 실용적인 서비스 제작 🚀

### 목표
- 복잡한 프로젝트 1개 완성
- 배포 준비
- 실전 경험 쌓기

### Day 15-17: 프로젝트 기획

**선택지:**
1. **블로그**: 글 작성, 댓글, 검색 기능
2. **포트폴리오**: 작품 전시, 연락 폼
3. **랜딩 페이지**: 제품 소개, 예약 기능

**기획 단계:**
```
1. 무엇을 만들지 결정
2. 필요한 기능 리스트 작성
3. 화면 구성 스케치 (손그림도 OK)
4. AI에게 전달할 요구사항 정리
```

**예시 (블로그 기획):**
```markdown
# 블로그 기획

## 필요한 페이지
- 홈: 최신 글 3개
- 글 목록: 모든 글 리스트
- 글 상세: 본문, 댓글
- 글 작성: 에디터

## 기능
- 마크다운 지원
- 태그 분류
- 검색 기능
- 다크 모드

## 디자인
- 미니멀한 느낌
- 흰색 배경, 검은색 텍스트
- 모바일 반응형
```

**소요 시간:** 2-3시간

---

### Day 18-20: 프로젝트 구현

**단계별 구현:**

**1단계: 기본 구조 (2시간)**
```
"Next.js로 블로그 프로젝트 시작해줘.
홈페이지, 글 목록, 글 상세 페이지 만들어줘."
```

**2단계: 기능 추가 (3시간)**
```
"마크다운 에디터 추가해줘. react-markdown 사용해서."
"글 목록에 태그 필터링 기능 추가해줘."
"검색 기능 만들어줘. 제목과 본문 검색되게."
```

**3단계: 디자인 개선 (2시간)**
```
"디자인을 더 세련되게 만들어줘.
- 타이포그래피 개선
- 여백 조정
- 부드러운 애니메이션 추가"
```

**4단계: 모바일 대응 (1시간)**
```
"모바일에서도 잘 보이게 반응형으로 만들어줘."
```

**소요 시간:** 8-10시간 (3일간)

### Day 21: 테스트 & 버그 수정

**테스트 체크리스트:**
- [ ] 모든 버튼 작동 확인
- [ ] 데이터 저장/불러오기 테스트
- [ ] 모바일에서 확인
- [ ] 다양한 브라우저에서 테스트 (Chrome, Safari, Firefox)
- [ ] 에러 처리 확인

**버그 수정:**
```
"[문제 상황] 버튼 클릭해도 반응 없어"
→ AI에게 에러 로그 보여주고 수정 요청

"[디자인 문제] 모바일에서 글자가 잘려"
→ "모바일에서 글자 잘리는 거 고쳐줘"
```

**소요 시간:** 2-3시간

**3주차 완료 기준:**
✅ 복잡한 프로젝트 1개 완성
✅ 모든 기능 정상 작동
✅ 모바일/데스크톱 대응 완료

### Day 22-23: 배포 준비

**체크리스트:**

- [ ] API 키 보안 처리
- [ ] 프로덕션 빌드 테스트
- [ ] 에러 로깅 설정

**배포 플랫폼 선택:**

**Vercel (추천):**
```
"Vercel에 배포할 준비 해줘"
→ AI가 설정 파일 자동 생성

vercel deploy
→ 클릭 몇 번으로 배포 완료
```

**Netlify:**
```
"Netlify 배포 설정 만들어줘"
→ GitHub 연동 후 자동 배포
```

**소요 시간:** 2-3시간

### Day 24-25: 성능 최적화

**최적화 항목:**

**1. 이미지 최적화**
```
"이미지를 WebP로 변환하고 lazy loading 적용해줘"
```

**2. 코드 스플리팅**
```
"페이지별로 코드 스플리팅 적용해줘"
```

**3. SEO 최적화**

**소요 시간:** 3-4시간
